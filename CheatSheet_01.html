  • Web Server

      • » A webserver shall store, process and deliver webpages to clients. The communication between the server and the client takes place using http. Mostly delivered are HTML Documents.

      » A web browser or a web crawler starts to communicate by making a request for a specific resource using HTTP or HTTPS. The server responds with the resource or an error.

      » But content can also be received from clients. This is used e.g. for web forms.

      » A lot of generic web servers nowadays also support server side scripting using a couple of scripting languages like PHP.

  • Web Client

      • » Also known as Web Browser, a web client is an application for accessing information on the world wide web. It fetches information resources and displays them to an end device like a computer.

      » The process is the following: The user inputs a URL, the rendering engine of the browser renders it and then displays it.

  • Hello World

      • » The on the head of a html document exists to show the browser the version of html that is used. It's not always required but you should still use it always.

      » The <body> element contains all of the contents such as text, images and other.

      » The <head> element contains all of the metadata and things like the <style> tag, the <script> tag or the <link> tag.

      » The <html> element shows the browser that the following document is a html document. It also is the root for all of the other tags and elements of the html document.

      » The <title> element is required in a html document, it is basically just the title of the document. It is between the <head> tags.
